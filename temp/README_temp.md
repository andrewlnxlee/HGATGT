# sim_env Directory Documentation

This directory contains the Python files for generating, training, evaluating, and visualizing simulated radar group tracking datasets using a Graph Neural Network (GNN) model (H-GAT-GT).

## File Descriptions

### 1. `config.py`
**Purpose:** Configuration file for dataset generation, model parameters, training hyperparameters, and output directories.
**Key Variables:**
- Data paths (`DATA_ROOT`).
- Number of samples and frames for generation (`NUM_TRAIN_SAMPLES`, `FRAMES_PER_SAMPLE`, etc.).
- Model configuration (`INPUT_DIM`, `EDGE_DIM`, `HIDDEN_DIM`).
- Training settings (`BATCH_SIZE`, `LEARNING_RATE`, `EPOCHS`, `MODEL_SAVE_PATH`).

### 2. `generate_data.py`
**Purpose:** Generates synthetic radar group tracking data simulating complex scenarios like converging (merging), diverging (splitting), and mixed interactions.
**How it works:**
- Defines a physical simulation engine (`ActiveInteractionScenarioEngine`) governing group movement.
- Spawns group leaders and members, applies guidance to targets, adds random wander and clutter, and handles splits and merges logically.
- Saves the sequence of point clouds with ground truth labels into `.npy` files for `train`, `val`, and `test` splits.

### 3. `dataset.py`
**Purpose:** Defines the PyTorch Geometric `Dataset` class (`RadarFileDataset`) to load the simulated data.
**How it works:**
- Reads `.npy` files generated by `generate_data.py`.
- For each frame, constructs a spatial graph by computing distances between points and connecting nearby points (Radius graph).
- Assigns edge labels based on whether two connected points belong to the same ground-truth group.
- Returns a list of `Data` objects representing the temporal sequence of a single simulation episode.

### 4. `model.py`
**Purpose:** Defines the architecture of the Graph Neural Network (GNN) model, `GNNGroupTracker`.
**Structure:**
- **FourierFeatureEncoder**: Enhances spatial awareness by mapping 2D coordinates to higher dimensions.
- **Node/Edge MLP**: Initial feature encoding.
- **Multi-Scale Backbone**: Uses 4 layers of `TransformerConv` for message passing.
- **AdaptiveLayerFusion**: Adaptively fuses features from the 4 layers instead of standard concatenation.
- **Decoders**:
  - Edge classifier predicts if an edge connects nodes in the same group.
  - Offset regressor predicts vector corrections to move points closer to their true cluster centroids and predicts uncertainty.

### 5. `train_sim.py`
**Purpose:** Script to train the `GNNGroupTracker` model.
**How it works:**
- Loads the datasets using PyTorch Geometric `DataLoader`.
- Computes a combined loss function (`compute_loss`):
  - Binary Cross-Entropy (BCE) for edge classification (with dynamic positive sample weighting).
  - Negative Log-Likelihood (NLL) for heteroscedastic regression of offsets.
- Uses AdamW optimizer with cosine annealing learning rate scheduler and gradient accumulation to handle graph batching limitations.
- Evaluates on the validation set and saves the best model.

### 6. `evaluate.py`
**Purpose:** Evaluates the trained H-GAT-GT model against various baseline tracking algorithms (Baseline DBSCAN+KF, GM-CPHD, CBMeMBer, Graph-MB).
**How it works:**
- Runs inference on the test dataset.
- Evaluates performance using standard Multi-Object Tracking metrics (MOTA, MOTP, IDSW) and clustering metrics (OSPA, G-IoU, Purity).
- Produces a comprehensive comparison DataFrame and saves a benchmark bar chart visualization (`final_benchmark_5way.png`).

### 7. `track_gif_gen.py`
**Purpose:** Visualizes the tracking performance of the trained model over time by generating an animated GIF.
**How it works:**
- Implements a `RobustGroupTracker` that takes GNN-clustered centroids and performs data association using the Hungarian algorithm and a constant velocity model.
- Handles track lifecycle management, splits, and merges.
- Plots points, tracks, group IDs, and trace tails over the sequence of frames, saving the result as a `.gif`.

### 8. `visualize_sim_dataset_trajectory.py`
**Purpose:** A utility script to visualize raw simulated trajectory data to verify dataset generation quality.
**How it works:**
- Loads a single test episode (`.npy`).
- Plots the accumulated positions of ground truth points over time using varying alpha transparency to represent temporal motion.
- Saves the resulting image as a `.jpg` file.

## Relationship Flow
1. **Configure**: Update settings in `config.py`.
2. **Generate**: Run `generate_data.py` to create the simulation dataset.
3. **Train**: Run `train_sim.py`, which uses `dataset.py` to load data and trains `model.py`.
4. **Evaluate/Visualize**:
   - `evaluate.py`: Compare trained model against baselines.
   - `track_gif_gen.py`: Create an animation of tracking performance.
   - `visualize_sim_dataset_trajectory.py`: Inspect the raw data.
